## ğŸŒ Deployment

### GitHub Pages (Recommended)

This documentation is configured for GitHub Pages deployment:

1. **Enable GitHub Pages** in your repository settings:
   - Go to Settings â†’ Pages
   - Source: Deploy from a branch
   - Branch: `gh-pages` (root)

2. **Deploy using GitHub Actions**:
   ```bash
   git add .
   git commit -m "Update documentation"
   git push
   ```

   The GitHub Actions workflow will automatically:
   - Build the documentation
   - Deploy to GitHub Pages
   - Make it available at `https://sparkison.github.io/m3u-editor-docs-v2/`

3. **Manual Deployment** (alternative):
   ```bash
   npm run deploy
   ```

### Custom Domain

To use a custom domain:

1. Add a `CNAME` file in `static/`:
   ```
   docs.yourdomainhere.com
   ```

2. Configure DNS:
   - Add a CNAME record pointing to `sparkison.github.io`

3. Update `docusaurus.config.js`:
   ```javascript
   url: 'https://docs.yourdomainhere.com',
   baseUrl: '/',
   ```

### Self-Hosting

To self-host the documentation:

1. **Build the site**:
   ```bash
   npm run build
   ```

2. **Serve the `build` directory** with any web server:

   **Nginx:**
   ```nginx
   server {
       listen 80;
       server_name docs.yourdomain.com;
       root /path/to/build;
       index index.html;
       
       location / {
           try_files $uri $uri/ /index.html;
       }
   }
   ```

   **Apache:**
   ```apache
   <VirtualHost *:80>
       DocumentRoot /path/to/build
       <Directory /path/to/build>
           AllowOverride All
           Require all granted
           RewriteEngine On
           RewriteCond %{REQUEST_FILENAME} !-f
           RewriteCond %{REQUEST_FILENAME} !-d
           RewriteRule ^ index.html [L]
       </RewriteRule>
   </VirtualHost>
   ```

   **Docker:**
   ```dockerfile
   FROM nginx:alpine
   COPY build /usr/share/nginx/html
   COPY nginx.conf /etc/nginx/conf.d/default.conf
   ```

### Vercel / Netlify

Both Vercel and Netlify can deploy Docusaurus sites:

**Vercel:**
1. Import repository
2. Build command: `npm run build`
3. Output directory: `build`

**Netlify:**
1. Import repository
2. Build command: `npm run build`
3. Publish directory: `build`

## ğŸ”§ Troubleshooting

### Build Errors

Clear cache and rebuild:
```bash
npm run clear
npm run build
```

### Port Already in Use

Change the port:
```bash
npm start -- --port 3001
```

### Broken Links

Check for broken links:
```bash
npm run build
```

Docusaurus will fail the build if there are broken links (configured with `onBrokenLinks: 'throw'`).
